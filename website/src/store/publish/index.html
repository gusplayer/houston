Publishing

Ship your agent to everyone running Houston

Three steps to get your agent into the Store. No SDKs, no servers, no review queue weeks. Just a public repo and a manifest.

1

Add a houston.json to your repo

The manifest is the source of truth. It tells Houston what your agent is, who built it, and which integrations it needs.

// houston.json
{
  "id": "your-agent-id",
  "name": "Your Agent",
  "description": "What it does in one or two sentences.",
  "version": "0.1.0",
  "author": "your-github-handle",
  "category": "marketing",
  "tags": ["email", "outreach"],
  "integrations": ["gmail", "notion"],
  "icon_url": "envelope"
}

Keep the id globally unique. Use lowercase, hyphens only. Bump version on every release so Houston can detect updates.

2

Push your repo to GitHub publicly

Houston downloads your agent straight from raw.githubusercontent.com. Your repo is the source of truth. No registry to push to, no artifact to upload.

Every change you commit ships to anyone with your agent installed the next time they hit Update. Public history, real authorship, zero lock in.

Recommended layout:

your-agent/
  houston.json
  CLAUDE.md           # system prompt
  .agents/
    skills/           # custom skills
  README.md
3

Submit to the catalog

The catalog index lives in our public repo. To get listed, send us the GitHub URL of your repo and a short note about what your agent does.

We are accepting early submissions by email. Soon this will be a self serve form right here on the site.
